# Operation Blitzsanierung

### Operation Summary
"Blitzsanierung" primary objective is refactoring the codebase, improving messy or slow code, fixing bugs,
and standardizing UI/UX. The operation is designed to be thorough but fast, to prepare for upcoming operations
where new features will be added or entire architecture will be changed.

### Operation Objectives
The operation will follow mission-style tactics. The codebase will be scouted for an issue, 
and then the issued will immediately be fixed via a commit, reducing time wasted and concetrating resourecs.

**Each objective will follow this format:**
- Mission 1: Scouting:
  Scout the targeted surface for issues
- Mission 2: Research:
  Research about the issues and find potential fixes
- Mission 3: Execution:
  Choose the optimal fix for the situation and apply
- Mission 4: Finalization:
  Apply finishing touches and polish, then commit to Git.

**Objective 1:** Fix user experience:
- **Operational Surface:** All .tsx files that are interacted with by end users
- **Known Issues:** Components are not standardized and are inconsistent, loading UX is terrible (inconsistent, glitchy)

**Objective 2:** Targeted Backend Refactoring:
- **Operational Surface:** Backend API routes which are used by the **chatting system** and **company dashboard system**
- **Known Issues:** Code quality issues, inconsitency between API routes.
